# Deployment AI Governance Standard (DAGS)
## Specification Status

Current Version: **v1.1**  
Status: **Published Standard**  
Effective Date: 2026-03-11

The Deployment AI Governance Standard (DAGS) defines governance requirements for the deployment and operational execution of AI-enabled systems.

DAGS governs the **deployment layer** of AI systems — the point at which models are integrated, configured, operated, and permitted to produce real-world effects.

The canonical specification is maintained by the **DAGS Governing Body**, which retains exclusive editorial, interpretive, and versioning authority.

---

## Current Release

The current published version is **DAGS v1.1**.

Version 1.1 introduces governance requirements for **agentic execution paths**, extending deployment-layer governance to sequences of consequential actions initiated by AI-enabled systems across multiple services and tools.

The five-domain architecture of the standard remains unchanged.

## Repository Structure

This repository contains the canonical artifacts associated with the DAGS standard.

- `SPEC/` Canonical normative specification
- `LEGAL/` Licensing and authority documents
- `RELEASES/` Detailed release notes
- `PROPOSALS/` Proposed changes under review

Normative requirements are defined within the **SPEC** directory.

Other documents in this repository are **non-normative supporting materials** unless explicitly designated otherwise.

---

## Authority Model

The DAGS Governing Body:

- maintains the canonical specification
- issues authoritative interpretations
- governs version releases
- licenses authorized service providers

No third party may issue authoritative interpretations, certifications, or compliance determinations unless explicitly authorized by the DAGS Governing Body.
